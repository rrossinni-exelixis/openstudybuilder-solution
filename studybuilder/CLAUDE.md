# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Documentation Structure

The documentation has been organized into focused files in `.claude/rules/` for better maintainability:

- **[overview.md](./.claude/rules/overview.md)** - Project overview and tech stack
- **[architecture-core.md](./.claude/rules/architecture-core.md)** - Core architecture, configuration, state management, and API layer
- **[commands.md](./.claude/rules/commands.md)** - Development commands and utilities
- **[routing-auth.md](./.claude/rules/routing-auth.md)** - Routing system and authentication flow
- **[components.md](./.claude/rules/components.md)** - Component structure, extensions, i18n, and styling
- **[patterns.md](./.claude/rules/patterns.md)** - Important patterns and best practices
- **[conventions-gotchas.md](./.claude/rules/conventions-gotchas.md)** - File naming conventions and common gotchas

Each file focuses on a specific aspect of the codebase, making it easier to find and update relevant information.
